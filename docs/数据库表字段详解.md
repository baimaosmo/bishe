# 数据库表字段详解

> 逐一说明 12 张数据库表每个字段的含义、数据类型、约束和业务用途。

---

## 一、users（普通注册用户表）

存储校外读者和系统管理员账号，使用「用户名 + 密码」方式登录。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTO_INCREMENT | 用户唯一ID，自增主键 |
| `username` | VARCHAR(50) | **UNIQUE**, NOT NULL | 登录用户名，全表唯一，如 `admin`、`reader001` |
| `email` | VARCHAR(120) | UNIQUE | 用户邮箱，可选但唯一，用于联系和找回密码 |
| `password_hash` | VARCHAR(255) | NOT NULL | 密码的哈希值（Werkzeug 生成），**不存明文** |
| `role` | VARCHAR(20) | DEFAULT 'user' | 角色：`user`（普通读者）或 `admin`（管理员）。管理员拥有后台管理权限 |
| `avatar` | VARCHAR(255) | — | 头像图片相对路径，如 `uploads/avatars/abc123.jpg` |

**代码中的方法**：
- `set_password(明文)` → 将明文密码哈希后存入 `password_hash`
- `check_password(明文)` → 将用户输入与数据库哈希值比对，返回 True/False

## 二、students（学生用户表）

存储校内学生账号，使用「学号 + 密码」方式登录。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTO_INCREMENT | 学生唯一ID |
| `student_no` | VARCHAR(20) | **UNIQUE**, NOT NULL | 学号，用于登录。如 `20210001` |
| `name` | VARCHAR(50) | NOT NULL | 学生真实姓名，如 `张明` |
| `gender` | VARCHAR(10) | — | 性别：男/女 |
| `major` | VARCHAR(80) | — | 专业名称，如 `计算机科学与技术` |
| `department` | VARCHAR(50) | DEFAULT '计算机学院' | 所属院系 |
| `password_hash` | VARCHAR(255) | NOT NULL | 密码哈希值，默认密码 `Student@123` |
| `avatar` | VARCHAR(255) | — | 头像路径 |

## 三、teachers（教师用户表）

存储校内教师账号，使用「工号 + 密码」方式登录。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTO_INCREMENT | 教师唯一ID |
| `job_no` | VARCHAR(20) | **UNIQUE**, NOT NULL | 工号，用于登录。如 `T202001` |
| `name` | VARCHAR(50) | NOT NULL | 教师真实姓名，如 `刘老师` |
| `gender` | VARCHAR(10) | — | 性别 |
| `major` | VARCHAR(80) | — | 专业/研究方向，如 `数据库技术` |
| `department` | VARCHAR(50) | DEFAULT '教务部门' | 所属院系/部门 |
| `password_hash` | VARCHAR(255) | NOT NULL | 密码哈希值，默认密码 `Teacher@123` |
| `avatar` | VARCHAR(255) | — | 头像路径 |

### 三张用户表设计对比

| 区别点 | User | Student | Teacher |
|--------|------|---------|---------|
| 登录凭证 | `username` 用户名 | `student_no` 学号 | `job_no` 工号 |
| 显示名称 | `username` | `name` | `name` |
| 独有字段 | `email`、`role` | `student_no`、`major` | `job_no`、`major` |
| 默认角色 | user/admin | student | teacher |
| 默认密码 | User@123 | Student@123 | Teacher@123 |

## 四、books（图书信息表）

存储馆藏图书的所有元数据。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTO_INCREMENT | 图书唯一ID |
| `title` | VARCHAR(100) | NOT NULL | 书名，如 `《Python程序设计基础》` |
| `author` | VARCHAR(50) | NOT NULL | 作者，如 `李华` |
| `publisher` | VARCHAR(100) | — | 出版社名称，如 `清华大学出版社` |
| `isbn` | VARCHAR(20) | **UNIQUE**, NOT NULL | ISBN 号，10位或13位数字，全表唯一，防止重复录入 |
| `category` | VARCHAR(50) | — | 图书分类标签，如 `计算机`、`人工智能`、`文学` |
| `floor` | INTEGER | DEFAULT 1 | 馆藏所在楼层（1/2/3/4） |
| `area` | VARCHAR(20) | DEFAULT 'A区' | 馆藏所在区域，如 `A区`、`B区` |
| `shelf` | VARCHAR(20) | DEFAULT '01架' | 书架号，如 `A-01` |
| `status` | VARCHAR(20) | DEFAULT 'available' | 借阅状态：`available`（可借）/ `borrowed`（已借出） |
| `description` | TEXT | — | 图书简介，长文本类型 |
| `cover_image` | VARCHAR(255) | — | 封面图片相对路径，如 `uploads/book_covers/xxx.jpg` |
| `add_time` | DATETIME | DEFAULT NOW | 图书入库时间 |

**状态流转**：
```
available ──借阅──▶ borrowed ──归还──▶ available
```

## 五、borrow_records（借阅记录表）

记录每一次图书借出和归还操作。**多态关联**：通过三个可空外键支持三种账号类型。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTO_INCREMENT | 记录唯一ID |
| `user_id` | INTEGER | FK→users.id, **NULLABLE** | 借阅者（普通用户），三选一 |
| `student_id` | INTEGER | FK→students.id, **NULLABLE** | 借阅者（学生），三选一 |
| `teacher_id` | INTEGER | FK→teachers.id, **NULLABLE** | 借阅者（教师），三选一 |
| `book_id` | INTEGER | FK→books.id, **NOT NULL** | 被借阅的图书 |
| `borrow_time` | DATETIME | DEFAULT NOW | 借阅时间 |
| `return_time` | DATETIME | **NULLABLE** | 实际归还时间，未归还时为 NULL |
| `status` | VARCHAR(20) | DEFAULT 'borrowing' | 记录状态：`borrowing`（借阅中）/ `returned`（已归还） |

**多态规则**：每条记录的 `user_id`、`student_id`、`teacher_id` **有且仅有一个非空**。代码中通过 `assign_current_account()` 自动填入当前用户对应的外键。

**ORM 关系**：
```python
record.user    → User 对象        # 通过 user_id 外键
record.student → Student 对象     # 通过 student_id 外键
record.teacher → Teacher 对象     # 通过 teacher_id 外键
record.book    → Book 对象        # 通过 book_id 外键
```

## 六、seats（座位信息表）

存储图书馆中每一个自习座位的属性。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTO_INCREMENT | 座位唯一ID |
| `floor` | INTEGER | NOT NULL, DEFAULT 1 | 所在楼层（1F/2F/3F） |
| `area` | VARCHAR(50) | NOT NULL | 所属区域，如 `自习A区`、`静音区`、`研讨间` |
| `seat_number` | VARCHAR(20) | **UNIQUE**, NOT NULL | 座位编号，全库唯一。如 `1F-A-01`、`E201` |
| `has_power` | BOOLEAN | DEFAULT FALSE | 是否有电源插座（True/False） |
| `status` | VARCHAR(20) | DEFAULT 'free' | 座位状态：`free`（空闲）/ `occupied`（已占用） |

**座位编号规则**：`楼层-区域前缀-序号`，如 `2F-J-03` = 2楼 J行 03号。

## 七、seat_reservations（座位预约记录表）

记录每一次座位预约和释放操作。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTO_INCREMENT | 预约记录唯一ID |
| `user_id` | INTEGER | FK→users.id, **NULLABLE** | 预约者（普通用户），三选一 |
| `student_id` | INTEGER | FK→students.id, **NULLABLE** | 预约者（学生），三选一 |
| `teacher_id` | INTEGER | FK→teachers.id, **NULLABLE** | 预约者（教师），三选一 |
| `seat_id` | INTEGER | FK→seats.id, **NOT NULL** | 被预约的座位 |
| `start_time` | DATETIME | DEFAULT NOW | 入座/预约时间 |
| `end_time` | DATETIME | **NULLABLE** | 离座时间，释放后才写入 |
| `status` | VARCHAR(20) | DEFAULT 'active' | 记录状态：`active`（使用中）/ `completed`（已结束） |

**计算属性 `account_name`**（不在数据库中存储，代码动态计算）：
```python
@property
def account_name(self):
    if self.user:    return self.user.username
    if self.student: return self.student.name
    if self.teacher: return self.teacher.name
    return '未知用户'
```

## 九、drift_books（漂流图书表）

存储用户发布的共享/交换图书信息。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTO_INCREMENT | 漂流图书唯一ID |
| `title` | VARCHAR(100) | NOT NULL | 书名，如 `《高等数学辅导书》` |
| `course_related` | VARCHAR(100) | — | 关联课程，如 `高等数学`、`考研英语` |
| `condition` | VARCHAR(20) | DEFAULT '良好' | 新旧程度：`全新` / `良好` / `一般` / `较旧` |
| `description` | TEXT | — | 补充描述或交换条件，如 `[免费赠送]适合大一复习` |
| `status` | VARCHAR(20) | DEFAULT 'drifting' | 漂流状态：`drifting`（漂流中）/ `claimed`（已领走） |
| `publish_time` | DATETIME | DEFAULT NOW | 发布时间 |
| `provider_user_id` | INTEGER | FK→users.id, **NULLABLE** | 提供者（普通用户），三选一 |
| `provider_student_id` | INTEGER | FK→students.id, **NULLABLE** | 提供者（学生），三选一 |
| `provider_teacher_id` | INTEGER | FK→teachers.id, **NULLABLE** | 提供者（教师），三选一 |
| `receiver_user_id` | INTEGER | FK→users.id, **NULLABLE** | 领取者（普通用户），三选一，领取后填入 |
| `receiver_student_id` | INTEGER | FK→students.id, **NULLABLE** | 领取者（学生），三选一 |
| `receiver_teacher_id` | INTEGER | FK→teachers.id, **NULLABLE** | 领取者（教师），三选一 |

**6 个外键分为两组**：
- **提供者组**（3个）：发布时填入，有且仅有一个非空
- **领取者组**（3个）：申请被接受后填入，接受前全为 NULL

**计算属性**：`provider_name` 和 `receiver_name` 自动根据非空外键返回对应用户的显示名。

**级联删除**：删除漂流图书时，自动删除关联的 `DriftRequest` 记录。

## 十、drift_requests（漂流图书领取申请表）

记录用户对漂流图书的领取请求。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTO_INCREMENT | 申请记录唯一ID |
| `book_id` | INTEGER | FK→drift_books.id, **NOT NULL** | 关联的漂流图书 |
| `message` | TEXT | — | 申请人给提供者的留言，如 `想借来复习` |
| `status` | VARCHAR(20) | DEFAULT 'pending' | 申请状态：`pending`（待确认）/ `accepted`（已同意）/ `rejected`（已拒绝） |
| `create_time` | DATETIME | DEFAULT NOW | 申请时间 |
| `receiver_user_id` | INTEGER | FK→users.id, **NULLABLE** | 申请者（普通用户），三选一 |
| `receiver_student_id` | INTEGER | FK→students.id, **NULLABLE** | 申请者（学生），三选一 |
| `receiver_teacher_id` | INTEGER | FK→teachers.id, **NULLABLE** | 申请者（教师），三选一 |

**状态流转**：
```
pending ──提供者同意──▶ accepted（同时图书变为claimed，其他申请变为rejected）
pending ──提供者拒绝──▶ rejected
```

## 十一、book_favorites（图书收藏表）

记录用户收藏了哪些图书。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTO_INCREMENT | 收藏记录唯一ID |
| `book_id` | INTEGER | FK→books.id, **NOT NULL** | 被收藏的图书 |
| `user_id` | INTEGER | FK→users.id, **NULLABLE** | 收藏者（普通用户），三选一 |
| `student_id` | INTEGER | FK→students.id, **NULLABLE** | 收藏者（学生），三选一 |
| `teacher_id` | INTEGER | FK→teachers.id, **NULLABLE** | 收藏者（教师），三选一 |
| `created_at` | DATETIME | DEFAULT NOW | 收藏时间 |

**级联删除**：删除图书时，自动删除该书的全部收藏记录（`cascade='all, delete-orphan'`）。

**业务规则**：
- 同一用户对同一本书只能收藏一次（应用层校验）
- 管理员不参与收藏功能
- 已收藏的书不会出现在个性化推荐中

## 十二、book_reviews（图书评价表）

存储用户对图书的评分和文字评价。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY, AUTO_INCREMENT | 评价记录唯一ID |
| `book_id` | INTEGER | FK→books.id, **NOT NULL** | 被评价的图书 |
| `user_id` | INTEGER | FK→users.id, **NULLABLE** | 评价者（普通用户），三选一 |
| `student_id` | INTEGER | FK→students.id, **NULLABLE** | 评价者（学生），三选一 |
| `teacher_id` | INTEGER | FK→teachers.id, **NULLABLE** | 评价者（教师），三选一 |
| `rating` | INTEGER | NOT NULL, DEFAULT 5 | 评分，1~5 星 |
| `content` | TEXT | NOT NULL | 评价文本内容，最少 5 个字 |
| `status` | VARCHAR(20) | DEFAULT 'pending' | 审核状态：`pending`（待审核）/ `approved`（已通过）/ `rejected`（已驳回） |
| `created_at` | DATETIME | DEFAULT NOW | 评价创建时间 |

**审核流程**：
```
用户提交 → status='pending' → 管理员点击AI审核 → DeepSeek判定
    ├─ approved → 在图书详情页公开展示
    └─ rejected → 不展示，仅在管理员后台可见
```

**业务规则**：
- 用户必须归还过该书才能评价
- 同一用户对同一本书只能评价一次
- 详情页只展示 `status='approved'` 的评价
- 平均评分只统计已通过的评价（`AVG(rating) WHERE status='approved'`）

## 字段命名规范速查

| 后缀/前缀 | 含义 | 示例 |
|----------|------|------|
| `_id` | 外键，关联其他表的主键 | `book_id`、`student_id` |
| `_time` | 时间戳，记录操作发生的时间 | `borrow_time`、`return_time` |
| `_at` | 创建时间 | `created_at`、`add_time` |
| `_hash` | 哈希值，不存明文 | `password_hash` |
| `_count` | 统计次数 | 无此字段（均为代码动态计算） |
| `has_` | 布尔值，表示"是否拥有" | `has_power` |
| `is_` | 布尔值，表示"是否为" | —（本项目未使用） |
| `status` | 状态枚举值 | `available/borrowed`、`free/occupied` |
