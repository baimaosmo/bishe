# 基于 Flask 的智慧图书馆管理系统的设计与实现

## 摘  要

随着高校信息化建设的不断推进，传统图书馆在图书借阅、座位管理、用户服务和数据统计等方面逐渐暴露出管理效率低、信息更新滞后、用户体验不足等问题。为提高图书馆资源管理效率，满足学生、教师及管理员对图书资源和学习空间的综合使用需求，本文设计并实现了一套基于 Flask 的智慧图书馆管理系统。

系统采用 B/S 架构，以 Python 语言和 Flask Web 框架为核心，使用 SQLAlchemy 作为 ORM 数据访问层，MySQL 作为后端数据库，前端采用 HTML、CSS、Jinja2 模板等技术完成页面展示与交互。系统围绕校园图书馆实际业务场景，设计了用户认证、图书管理、图书借阅、座位预约、图书漂流、个人中心、用户管理、图书评价、个性化图书推荐、AI 智能检索和数据统计看板等功能模块。学生和教师等读者用户可以登录系统查询图书、借阅归还图书、收藏图书、发布评价、查看系统推荐书单、预约座位和参与图书漂流；管理员可以维护馆藏图书、管理用户、审核评价、查看借阅与座位使用数据，从而实现图书馆业务流程的信息化和集中化管理。

系统实现过程中重点解决了多类型账号统一认证、多角色权限控制、图书状态同步、座位预约超时释放、图书评价审核、漂流图书申请处理、基于借阅与收藏行为的个性化推荐以及基于本地馆藏数据的 AI 智能检索等问题。测试结果表明，该系统功能完整、操作便捷、结构清晰，能够满足校园图书馆日常管理和用户服务的基本需求，具有一定的实用价值和扩展空间。

**关键词：**智慧图书馆；Flask；MySQL；图书管理；座位预约；AI 智能检索

## Abstract

With the continuous development of campus informatization, traditional library management has gradually exposed problems such as low management efficiency, delayed information updates and insufficient user experience in book borrowing, seat management and data statistics. In order to improve the efficiency of library resource management and meet the comprehensive needs of students, teachers and administrators, this paper designs and implements a smart library management system based on Flask.

The system adopts the B/S architecture. Python and Flask are used as the main development technologies, SQLAlchemy is used as the ORM data access layer, and MySQL is used as the database. The front-end pages are implemented with HTML, CSS and Jinja2 templates. According to the actual library business process, the system includes user authentication, book management, borrowing management, seat reservation, book crossing, personal center, user management, review management, personalized book recommendation, AI intelligent search and data dashboard modules. Students and teachers can search books, borrow and return books, collect books, write reviews, view recommended book lists, reserve seats and participate in book crossing. Administrators can maintain book information, manage users, review comments and view statistical data.

During the implementation, the system focuses on multi-account authentication, role-based access control, book status synchronization, automatic release of expired seat reservations, review moderation, book crossing application processing, personalized recommendation based on borrowing and collection behavior, and AI intelligent search based on local book data. The test results show that the system has complete functions, clear structure and convenient operation, and can meet the basic needs of campus library management and user services.

**Key words:** Smart Library; Flask; MySQL; Book Management; Seat Reservation; AI Search

## 第 1 章 绪论

### 1.1 课题背景

高校图书馆是校园学习和教学活动的重要基础设施，不仅承担图书资料的收藏、检索和借阅服务，还承担学习空间管理、阅读推广和信息服务等职责。随着在校师生规模不断扩大，图书馆的服务对象、馆藏数量和管理数据不断增加，传统依赖人工登记或分散表格管理的方式已经难以满足实际需求。

在传统图书馆管理模式中，管理员需要手动维护图书信息、借阅状态和读者信息，容易出现数据重复、统计困难、状态更新不及时等问题。用户在查询图书、了解馆藏位置、查看借阅记录和预约座位时，也常常需要依赖线下咨询或人工处理，影响使用效率。与此同时，校园用户对智能化服务的需求也在提升，例如希望系统能够根据馆藏数据进行图书推荐、能够查看图书馆座位实时状态、能够在线参与闲置教材和书籍流转等。

因此，开发一套面向校园场景的智慧图书馆管理系统具有较强的现实意义。该系统通过 Web 平台将图书管理、借阅管理、座位预约、图书漂流和数据统计等功能整合到统一平台中，有助于提升管理员工作效率，改善用户使用体验，并为图书馆后续智能化建设提供基础。

### 1.2 课题研究意义

本课题的研究意义主要体现在以下几个方面：

第一，提高图书馆管理效率。系统将图书信息录入、查询、借阅、归还、删除和修改等操作进行信息化管理，减少人工登记带来的错误，提高馆藏资源维护效率。

第二，优化用户服务体验。用户登录系统后可以在线检索图书、查看图书详情、收藏感兴趣的图书、查看个人借阅记录，还可以进行座位预约和图书漂流操作，减少线下排队和人工沟通成本。

第三，提升资源利用率。座位预约模块可以展示座位空闲状态并自动释放超时座位，图书漂流模块可以促进闲置书籍在师生之间流转，个性化推荐模块可以根据用户借阅和收藏偏好引导用户发现更多馆藏资源，提高学习资源复用率。

第四，辅助管理决策。数据看板能够统计馆藏数量、借阅趋势、热门分类和座位使用情况，为管理员调整馆藏结构、优化座位管理和制定服务策略提供参考。

### 1.3 国内外研究现状

国外高校图书馆信息化建设起步较早，许多高校已经建设了较成熟的数字图书馆平台和集成化图书馆管理系统。这类系统通常支持 OPAC 检索、电子资源访问、借阅管理、读者服务、馆藏统计和移动端访问等功能，部分系统还结合推荐算法、RFID、自助借还设备和空间预约系统，提高资源管理和用户服务水平。

国内高校图书馆近年来也在不断推进数字化与智慧化建设。许多高校已经实现图书检索、在线续借、电子资源访问、座位预约和入馆预约等功能。但对于中小型院校或课程设计、毕业设计场景而言，现有大型系统往往部署复杂、成本较高，且不一定适合教学实践和二次开发。因此，基于轻量级 Web 框架设计一套结构清晰、功能完整、易于扩展的智慧图书馆管理系统，具有良好的学习和应用价值。

本课题基于 Flask 框架实现校园智慧图书馆管理系统，重点关注系统业务流程的完整性、功能模块的可用性和数据库结构的合理性，在满足基本图书馆管理需求的同时，加入了座位预约、图书漂流、个性化图书推荐、AI 智能检索和数据统计功能，使系统更符合现代校园图书馆的综合服务需求。

## 第 2 章 相关技术介绍

### 2.1 Python 语言

Python 是一种语法简洁、可读性较强的高级程序设计语言，具有开发效率高、第三方库丰富和跨平台能力强等特点。在 Web 应用开发中，Python 可以结合 Flask、Django 等框架快速构建业务系统，也可以通过 pandas、requests 等库完成数据处理和接口调用。本系统以 Python 作为后端主要开发语言，用于实现用户认证、图书借阅、座位预约、图书漂流、推荐计算、数据统计以及 AI 接口调用等业务逻辑。Python 清晰的语法结构也有利于系统后期维护和功能扩展。

### 2.2 Flask 框架

Flask 是 Python 生态中常用的轻量级 Web 框架，核心功能简洁，扩展方式灵活，适合中小型管理系统和教学实践项目开发。Flask 通过路由机制接收并处理浏览器请求，通过模板引擎返回动态页面，也可以结合 Session 实现登录状态保持。本系统采用 Flask 作为 Web 后端框架，并使用蓝图机制对功能进行模块化划分，将首页、用户认证、图书管理、座位预约、AI 工具和图书漂流等业务分别组织在不同路由模块中。这样的设计降低了单个文件的复杂度，使系统结构更加清晰，也便于后续增加新的功能模块。

### 2.3 MySQL 数据库

MySQL 是一种常用的关系型数据库管理系统，具有运行稳定、查询能力强、支持事务处理和数据约束等优点，适合保存结构化业务数据。智慧图书馆管理系统涉及用户信息、学生信息、教师信息、馆藏图书、借阅记录、座位信息、预约记录、图书收藏、图书评价、漂流图书和领取申请等多类数据，这些数据之间存在较明确的关联关系。例如，借阅记录需要关联具体用户和图书，座位预约记录需要关联预约用户和座位，图书评价需要关联评价用户和被评价图书。

本系统使用 MySQL 作为后端数据库，通过主键、外键和字段约束维护数据之间的关系，保证业务数据的完整性和一致性。在图书借阅、归还、座位预约和释放等操作中，系统需要同步更新多张表中的状态信息，MySQL 的事务能力能够为这些关键业务操作提供可靠的数据支持。同时，MySQL 对条件查询、模糊检索、分组统计和排序等操作支持较好，能够满足系统中图书检索、借阅统计、座位统计和推荐数据计算等需求。

### 2.4 SQLAlchemy ORM

SQLAlchemy 是 Python 中常用的对象关系映射框架，可以将数据库表映射为 Python 类，将表中的记录映射为对象。使用 ORM 后，开发者可以通过操作对象的方式完成数据增删改查，减少直接编写 SQL 语句的工作量，并提高代码的可维护性。本系统通过 SQLAlchemy 定义 User、Student、Teacher、Book、BorrowRecord、Seat、SeatReservation、DriftBook、DriftRequest、BookFavorite、BookReview 等模型类，将系统核心业务表统一组织在模型层中。

在具体实现中，SQLAlchemy 不仅用于普通的数据查询和保存，还用于建立模型之间的关联关系。例如，借阅记录可以通过关系字段获取对应图书，座位预约记录可以关联具体座位，图书漂流申请可以关联漂流图书。通过 ORM 关系映射，系统页面在展示借阅记录、预约记录和漂流申请时能够更方便地获取关联数据，减少重复查询逻辑。

### 2.5 Jinja2 模板与前端技术

Jinja2 是 Flask 默认集成的模板引擎，支持变量输出、条件判断、循环遍历、模板继承和宏定义等功能，适合生成动态 Web 页面。本系统使用 Jinja2 将后端查询得到的数据渲染到前端页面中，例如图书列表、图书详情、我的借阅、座位大厅、漂流广场、用户管理、互动审核和推荐书单等页面。通过模板继承机制，系统可以在统一的基础布局中复用导航栏、页面结构和公共样式，减少重复代码。

前端部分主要使用 HTML、CSS 和少量页面交互实现系统界面。系统通过表单提交完成登录、注册、图书添加、评价发布、漂流申请等操作，通过按钮和链接触发借阅、归还、收藏、预约和释放座位等业务。页面样式围绕管理系统的使用场景进行设计，重点保证功能入口清晰、信息层次明确和操作反馈直观。

### 2.6 AI 接口与推荐技术

系统包含两类与智能化服务相关的实现：一类是基于本地数据计算的图书推荐，另一类是基于外部大模型接口的 AI 智能检索。图书推荐功能主要根据用户历史借阅分类、收藏分类、全馆借阅次数、收藏热度和最新入库时间生成推荐书单。该方式不依赖复杂算法模型，但能够结合用户行为和全馆热度，为用户提供较有参考价值的阅读建议。

AI 智能检索功能调用 DeepSeek API，实现基于自然语言的图书推荐和报告生成。系统在调用接口前会读取数据库中的真实馆藏图书信息，并将书名、作者、分类和状态等内容组织为上下文，再与用户问题一起发送给 AI 接口。通过这种方式，AI 回复能够围绕本地图书馆实际馆藏展开，减少推荐不存在图书的情况，提高智能检索结果与系统数据的一致性。

### 3.1 可行性分析

#### 3.1.1 技术可行性

本系统采用 Python、Flask、SQLAlchemy、MySQL、Jinja2、HTML 和 CSS 等技术完成开发，这些技术成熟稳定，资料丰富，能够满足校园图书馆管理系统的开发需求。Flask 框架适合构建轻量级 Web 应用，蓝图机制能够将不同业务功能拆分为独立模块；SQLAlchemy 能够将数据库表映射为模型类，降低数据访问复杂度；MySQL 能够稳定保存用户、图书、借阅、座位和互动等结构化数据。

从系统功能角度看，图书检索、借阅归还、座位预约、用户管理、图书漂流、个性化推荐和数据统计等功能都可以通过现有技术实现。AI 智能检索功能通过调用外部 DeepSeek API 完成，自身系统只需负责组织本地馆藏上下文、发送请求和展示结果。因此，系统整体具备较高的技术可行性。

从管理成本角度看，系统上线后可以将图书录入、借阅归还、座位预约、用户管理、评价审核和数据统计等工作集中到统一平台中处理，减少人工登记、纸质记录和重复统计带来的时间成本。对于校园图书馆或教学实践场景而言，该系统能够在较低投入下提升管理效率和服务质量，因此具有较好的经济可行性。

#### 3.1.3 社会可行性

随着高校数字化校园建设不断推进，师生已经普遍习惯通过 Web 系统完成选课、查询、预约和信息管理等操作，因此基于浏览器的图书馆管理系统符合当前校园信息化使用习惯。系统将图书检索、借阅归还、座位预约、图书漂流、个人中心和后台管理等功能整合到统一平台中，有助于提升图书馆服务效率，减少线下排队和人工沟通成本。

从用户角度看，学生和教师等读者用户可以更加便捷地获取馆藏资源、预约学习座位、查看推荐书单并参与闲置图书流转；从管理员角度看，系统能够帮助其更加规范地维护图书、用户和借阅数据，并通过统计信息辅助管理决策。系统功能围绕校园学习资源共享和图书馆日常管理展开，不涉及违反法律法规或社会伦理的问题，具有积极的应用价值。因此，该系统具备较好的社会可行性。

### 3.2 用户需求分析

根据校园图书馆系统的使用对象和业务权限，系统用户主要分为读者用户和管理员用户两大类。其中读者用户包括学生用户和教师用户，二者均可在注册页面填写学号或工号、姓名、性别、专业和密码完成账号开通，也可以由管理员批量导入学生账号。管理员用户主要负责后台数据维护、权限控制和业务审核。不同用户在系统中的操作范围不同，具体需求如下。

1. 读者用户需求描述：
（1）学生用户可以使用学号注册和登录，教师用户可以使用工号注册和登录，注册信息包括账号编号、姓名、性别、专业和密码。
（2）读者用户能够查看馆藏图书列表，按照书名、作者、ISBN 或分类检索图书信息。
（3）读者用户可以进入图书详情页面，查看图书简介、出版社、馆藏位置、借阅状态、评分和评价等信息。
（4）读者用户可以借阅当前状态为可借的图书，并在个人借阅记录中查看借阅和归还情况。
（5）读者用户可以归还本人已借阅的图书，归还后可以对图书进行评分和评价。
（6）读者用户可以收藏感兴趣的图书，并在我的收藏页面查看或取消收藏。
（7）读者用户可以查看系统根据借阅、收藏、热度和新书数据生成的推荐书单。
（8）读者用户可以进入选座管理页面，按楼层查看图书馆区域布局、座位状态、电源座位和当前预约情况，预约空闲座位并在使用结束后释放座位。
（9）读者用户可以参与图书漂流，发布闲置教材或课外书，也可以申请领取其他用户发布的图书，并处理本人发布图书的领取申请。
（10）读者用户可以查看个人中心，了解借阅、收藏、评价、座位和漂流参与情况，也可以上传头像并修改登录密码。
（11）读者用户可以使用 AI 智能搜索功能，通过自然语言描述阅读需求，由系统结合本地馆藏数据生成推荐结果并支持多轮追问。

2. 管理员用户需求描述：

（2）管理员可以进行图书管理，包括新增图书、编辑图书信息、上传图书封面和删除符合条件的图书。
（3）管理员可以管理出版社、图书分类、馆藏位置等图书基础信息，保证馆藏数据规范完整。
（4）管理员可以查看学生用户、教师用户以及系统管理员账号信息，并对用户进行密码重置或删除等管理操作。
（5）管理员可以批量导入学生账号信息，导入字段包括学号、姓名、性别和专业，便于校内学生统一使用学号登录系统。
（6）管理员可以查看图书借阅记录、座位预约记录、热门收藏图书和用户互动数据。
（7）管理员可以审核用户提交的图书评价，决定评价是否在图书详情页公开展示。
（8）管理员可以管理图书漂流信息，包括查看、编辑或删除漂流图书和相关申请记录。
（9）管理员可以查看数据统计看板，了解近 30 天借阅趋势、图书分类分布、座位使用热力和系统运行概况。
（10）管理员可以使用 AI 报告生成功能，根据馆藏、借阅和座位统计数据生成运营分析报告。
（11）管理员可以根据系统统计结果调整馆藏维护、座位管理和图书服务策略。

### 3.3 系统用例分析

系统用例分析是在用户需求分析的基础上，对不同参与者与系统功能之间的交互关系进行抽象描述。通过用例分析，可以明确系统需要向不同用户提供哪些服务、各项服务的触发条件和基本处理结果，为后续系统总体设计、数据库设计和功能实现提供依据。

本智慧图书馆管理系统面向校园图书馆的日常服务与管理场景，主要参与者包括读者用户和管理员用户。读者用户由学生用户和教师用户组成，主要围绕图书检索、图书借阅、座位预约、图书互动、AI 智能检索和个人信息维护等业务进行操作；管理员属于管理端用户，主要负责馆藏数据维护、用户账号维护、学生账号导入、互动内容审核、漂流图书管理以及数据统计分析等工作。系统在用例设计中根据账号类型和角色权限进行访问控制，保证不同用户只能访问与其身份相匹配的功能。

#### 3.3.1 参与者分析

根据系统业务范围和权限差异，系统参与者可划分为三类，具体如下。

| 参与者 | 身份说明 | 主要操作范围 |
| --- | --- | --- |
| 学生用户 | 使用学号注册或由管理员批量导入的校内学生 | 图书查询、借阅归还、收藏评价、推荐查看、座位预约、图书漂流、AI 搜索、头像和密码维护 |
| 教师用户 | 使用工号注册并登录的校内教师或图书馆工作人员 | 图书查询、借阅归还、收藏评价、推荐查看、座位预约、图书漂流、AI 搜索、个人信息维护 |
| 管理员 | 具有后台管理权限的系统维护人员 | 图书管理、用户管理、学生导入、评价审核、漂流管理、借阅与座位数据查看、数据大屏与报告生成 |

从参与者关系来看，读者端用户是系统使用频率最高的群体，其核心目标是便捷获取馆藏资源、完成图书借还和座位使用；管理员是系统正常运行的保障者，其核心目标是维护数据准确性、处理用户互动内容并掌握图书馆运行状态。系统需要同时满足读者端的便捷性和管理端的规范性。

#### 3.3.2 读者端用户用例分析

读者端用户包括学生用户和教师用户。两类用户登录方式有所不同，学生使用学号登录，教师使用工号登录，但进入系统后所能使用的图书馆服务功能基本一致。读者端主要用例包括账号注册与登录、浏览图书、检索图书、查看图书详情、借阅图书、归还图书、查看借阅记录、收藏图书、取消收藏、提交图书评价、查看推荐书单、预约座位、释放座位、发布漂流图书、申请领取漂流图书、处理本人发布图书的领取申请、AI 智能搜索、上传头像和修改密码等。

读者进入系统后，首先需要完成身份认证。认证通过后，系统根据账号类型记录当前用户身份，并在后续借阅记录、座位预约记录、收藏记录、评价记录和漂流记录中写入对应的用户标识。用户在图书列表页面可以根据书名、作者、ISBN 或分类进行检索，进入详情页后可查看图书封面、简介、出版社、馆藏位置、当前状态、平均评分和已审核评价。当图书状态为可借时，用户可以发起借阅操作，系统生成借阅记录并同步更新图书状态；当用户完成阅读后，可以在我的借阅页面归还图书，归还成功后图书重新变为可借状态。

除基本借还功能外，读者端还提供个性化和互动服务。用户可以收藏感兴趣的图书，系统根据收藏分类、借阅分类、借阅热度、收藏热度和新书入库时间生成推荐书单。用户归还图书后可以提交评分和文字评价，评价经管理员审核通过后在图书详情页展示。座位预约用例用于解决图书馆阅览座位占用状态不透明的问题，用户可以按楼层查看座位状态，预约空闲座位并在离开时主动释放。图书漂流用例用于支持闲置教材和课外书共享，用户既可以发布自己的漂流图书，也可以对他人发布的图书提交领取申请，图书提供者可对申请进行同意或拒绝处理。

#### 3.3.3 管理端用户用例分析

管理员是系统后台管理功能的主要参与者，承担数据维护、权限控制和业务监督职责。管理员登录后可以进入具有管理权限的页面，完成图书信息维护、用户账号维护、学生账号批量导入、图书评价审核、用户互动数据查看、图书漂流信息管理和数据统计分析等操作。

在图书管理方面，管理员可以新增图书并填写 ISBN、书名、作者、出版社、分类、楼层、区域、书架、简介和封面等信息。系统需要对 ISBN 进行格式校验，并限制重复录入，保证馆藏数据的一致性。管理员还可以修改已有图书信息，删除未处于借出状态的图书，维护出版社等基础数据。在用户管理方面，管理员可以查看学生用户、教师用户和管理员账号信息，对用户进行密码重置或删除操作，并通过表格批量导入学生账号，提高校内账号维护效率。

在业务监督方面，管理员可以查看用户提交的图书评价，并对评价执行通过或拒绝操作，避免不合规内容直接展示在图书详情页。管理员还可以查看借阅记录、座位预约记录、热门收藏图书和近期互动情况，从而了解系统使用状态。对于图书漂流业务，管理员能够查看、编辑或删除漂流图书及相关申请记录，保证图书共享流程规范运行。数据统计看板则帮助管理员掌握近 30 天借阅趋势、图书分类分布、座位预约热力和座位占用率等信息，为馆藏补充和座位管理提供参考。

### 3.4 功能需求分析

#### 3.4.1 用户认证需求

系统需要支持多类型账号统一登录，包括学生用户、教师用户和管理员用户。学生通过学号登录，教师通过工号登录，管理员通过管理员账号登录。登录成功后，系统需要记录当前账号的 ID、账号类型、用户名、角色和头像等信息，用于后续页面展示、数据筛选和权限判断。

系统还需要支持学生和教师用户注册、退出登录、修改密码和上传头像。注册时学生填写学号、姓名、性别、专业和密码，教师填写工号、姓名、性别、专业和密码。为方便学校集中维护学生信息，管理员应能够通过 Excel 或 CSV 表格批量导入学生数据，并能够在用户管理页面查看用户信息、重置用户密码或删除无效用户。密码修改和重置后应使用哈希方式保存，避免明文密码存储带来的安全风险。

#### 3.4.2 图书管理需求

图书管理是系统的基础功能。管理员需要能够录入图书的 ISBN、书名、作者、出版社、分类、馆藏楼层、区域、书架、简介、封面图片和借阅状态等信息。系统应对 ISBN 进行格式校验，并限制重复 ISBN，保证馆藏数据的规范性。管理员还应能够修改图书信息、更新封面图片和删除不再馆藏的图书。

学生和教师需要能够浏览图书列表，根据书名、作者、ISBN 或分类快速检索图书，并进入详情页查看图书介绍、馆藏位置、当前状态、评分和评价内容。图书处于可借状态时，用户可以发起借阅；图书处于借出状态时，系统应明确显示不可借状态。

#### 3.4.3 借阅管理需求

系统需要支持完整的借阅和归还流程。用户借阅图书时，系统应判断图书是否存在、当前是否可借，并在借阅成功后生成借阅记录，同时将图书状态更新为已借出。用户可以在“我的借阅”页面查看个人历史借阅记录，包括借阅时间、归还时间、图书信息和当前状态。

用户归还图书时，系统应校验该借阅记录是否属于当前用户，避免用户操作他人的借阅记录。归还成功后，系统需要将借阅记录状态更新为已归还，记录归还时间，并将图书状态恢复为可借。管理员可以在互动管理或数据统计页面查看借阅情况，为馆藏维护和数据分析提供依据。

#### 3.4.4 座位预约需求

系统需要提供阅览座位可视化管理功能。用户进入座位页面后，可以按楼层查看图书馆平面布局、区域位置、中心通道、入口、窗边和书架等空间信息，并了解座位号、是否有电源、空闲或占用状态等信息。用户只能预约当前空闲的座位，预约成功后系统需要生成座位预约记录，并将座位状态更新为占用。

为避免恶意占座和重复占座，系统需要限制同一用户同一时间只能存在一个有效座位预约。用户可以主动释放座位，释放后预约记录结束，座位状态恢复为空闲。系统还需要自动检测超过 3 小时的有效预约并释放座位，同时在预约即将到期时给出提示。座位页面应展示当前楼层座位总数、空闲数、占用数和占用率，便于用户选择合适区域。

#### 3.4.5 图书收藏、评价与推荐需求

系统需要支持图书收藏和评价功能。用户可以收藏感兴趣的图书，并在“我的收藏”页面查看收藏列表，也可以取消收藏。用户归还图书后可以提交评分和文字评价，系统需要限制未归还图书不能评价、同一用户不能重复评价同一本书。评价提交后默认进入待审核状态，管理员审核通过后才能在图书详情页展示。

在收藏和借阅数据基础上，系统还需要提供图书推荐功能。推荐页面应综合用户历史借阅分类、收藏分类、全馆借阅次数、收藏热度和最新入库时间生成推荐书单。当用户已有借阅或收藏记录时，系统应优先展示符合个人偏好的图书；当用户行为数据不足时，系统应通过借阅榜、收藏热度和新书推荐补充推荐结果。推荐过程中应尽量排除用户已经借阅或收藏过的图书，提升推荐内容的新鲜度和实用性。

#### 3.4.6 图书漂流需求

系统需要提供图书漂流角功能，用于促进闲置书籍在校园用户之间流转。用户可以发布漂流图书，填写书名、关联课程、新旧程度、交换方式和补充说明。其他用户可以在漂流广场中按照状态、课程或关键词筛选图书，并进入详情页查看图书信息。

用户可以对漂流中的图书提交领取申请，并填写留言。系统需要限制用户不能申请自己发布的图书，也不能对同一本书重复提交申请。图书提供者或管理员可以同意或拒绝申请。申请通过后，漂流图书状态更新为已领走，并记录领取者信息；同一本书的其他申请应自动更新为拒绝状态，保证漂流流程结果唯一。

#### 3.4.7 AI 智能工具与数据统计需求

系统需要提供 AI 智能检索功能。用户可以使用自然语言描述阅读需求，系统根据本地馆藏数据组织提示词并调用外部大模型接口，返回与馆藏图书相关的推荐结果。该功能应尽量围绕系统中真实存在的图书进行回答，减少虚构图书信息。

系统还需要提供报告生成和数据统计功能。管理员可以查看近 30 天借阅趋势、图书分类借阅分布、座位预约热力数据、馆藏总数、累计借阅数、当前借阅数和座位占用率等内容。系统根据统计结果生成简要分析建议，辅助管理员了解图书馆资源使用情况和管理重点。

### 3.5 非功能需求分析

1. 安全性：系统需要对用户密码进行哈希存储，避免明文保存；管理员功能必须进行权限校验；读者用户只能操作自己的借阅、预约、收藏和漂流申请数据；上传头像和图书封面时需要限制文件类型，降低非法文件上传风险。
2. 易用性：系统页面应按业务场景组织功能入口，图书检索、借阅、归还、预约、收藏和申请等操作应尽量简洁；操作成功或失败后应给出明确提示，帮助用户理解处理结果。
3. 数据一致性：图书借阅和归还需要同步更新借阅记录与图书状态，座位预约和释放需要同步更新预约记录与座位状态，漂流申请通过后需要同步更新图书状态和其他申请状态，避免业务数据相互矛盾。
4. 可维护性：系统应采用模块化结构组织代码，路由、模型和模板职责清晰，便于后续维护。数据库模型应集中定义，公共的账号筛选、账号绑定和文件上传逻辑应尽量复用。
5. 性能需求：系统应能够支持常见的图书列表查询、分类筛选、模糊搜索、借阅统计和推荐计算。对于列表类数据，后续可通过分页和索引优化提升大量数据场景下的响应速度。
6. 扩展性：系统后续应便于扩展移动端适配、借阅到期提醒、续借功能、座位分时段预约、统计报表导出和更复杂的推荐算法。

## 第 4 章 系统总体设计

### 4.1 系统设计目标

本系统设计目标是实现一个面向校园图书馆场景的综合管理平台，使管理员能够高效维护图书馆资源，使用户能够便捷使用图书馆服务。系统应达到以下目标：

1. 实现图书信息的统一维护和快速检索。
2. 实现用户借阅、归还和个人记录查询。
3. 实现阅览座位的可视化预约与释放。
4. 实现学生、教师和管理员的多角色管理。
5. 实现图书漂流和用户互动功能，提高资源流转效率。
6. 实现数据统计、个性化推荐和智能检索，提升系统智能化水平。

### 4.2 系统架构设计

系统采用典型 B/S 架构，用户通过浏览器访问系统页面，浏览器向 Flask 后端发送 HTTP 请求。Flask 后端根据路由分发请求，调用业务逻辑和数据库模型完成处理，再通过 Jinja2 模板生成 HTML 页面返回给用户。

系统整体可分为表现层、业务逻辑层和数据访问层：

表现层：由 HTML、CSS 和 Jinja2 模板组成，负责页面展示、表单输入、列表展示和操作按钮。

业务逻辑层：由 Flask 蓝图路由组成，负责处理登录认证、图书管理、借阅、座位预约、漂流申请、评价审核、推荐计算和 AI 调用等业务逻辑。

数据访问层：由 SQLAlchemy 模型和 MySQL 数据库组成，负责持久化保存用户、图书、借阅记录、座位、评价和漂流图书等数据。

### 4.3 功能模块设计

系统主要划分为以下模块：

1. 用户认证模块：负责学生和教师注册、登录、退出、修改密码、头像上传、学生导入和用户管理。
2. 图书管理模块：负责图书列表、图书检索、图书添加、图书编辑、图书删除和图书详情展示。
3. 借阅管理模块：负责图书借阅、归还和个人借阅记录查询。
4. 座位预约模块：负责座位状态展示、座位预约、座位释放和超时自动释放。
5. 互动评价模块：负责图书收藏、评价提交、评价审核和热门图书统计。
6. 图书漂流模块：负责漂流图书发布、浏览、申请、申请处理和管理员维护。
7. 推荐与 AI 工具模块：负责个性化图书推荐、智能检索、报告生成和数据分析展示。
8. 首页看板模块：负责展示系统统计数据和最近活动动态。

### 4.4 数据库设计

系统数据库围绕用户、图书、借阅、座位和互动数据进行设计，主要数据表如下：

#### 4.4.1 用户相关表

用户相关表包括 users、students 和 teachers。users 表主要保存管理员账号和历史兼容账号，主要字段包括 id、username、email、password_hash、role 和 avatar。students 表保存学生信息，主要字段包括 id、student_no、name、gender、major、department、password_hash 和 avatar。teachers 表保存教师信息，主要字段包括 id、job_no、name、gender、major、department、password_hash 和 avatar。

系统采用多账号表设计，能够分别适配管理员、学生和教师等不同登录方式。各类业务记录通过 user_id、student_id 和 teacher_id 三个可空外键字段关联到不同账号类型，其中读者端业务主要关联 student_id 或 teacher_id。

#### 4.4.2 图书相关表

books 表保存馆藏图书基础信息，主要字段包括 id、title、author、publisher、isbn、category、floor、area、shelf、status、description、cover_image 和 add_time。publisher 表保存出版社数据，便于管理员录入图书时复用已有出版社名称。

borrow_records 表保存借阅记录，主要字段包括 id、user_id、student_id、teacher_id、book_id、borrow_time、return_time 和 status。该表用于记录每次借阅和归还过程。

book_favorites 表保存用户收藏图书记录，book_reviews 表保存用户评价记录，其中评价表包含 rating、content、status 和 created_at 等字段，用于支持评分、评论和审核。

#### 4.4.3 座位相关表

seats 表保存座位基础信息，主要字段包括 id、floor、area、seat_number、has_power 和 status。seat_reservations 表保存座位预约记录，主要字段包括 id、user_id、student_id、teacher_id、seat_id、start_time、end_time 和 status。

座位预约记录与座位表关联，系统通过预约记录状态和座位状态共同反映当前座位使用情况。

#### 4.4.4 图书漂流相关表

图书漂流功能主要包含 drift_books 和 drift_requests 两张表。drift_books 表保存用户发布的漂流图书信息，包括书名、关联课程、新旧程度、描述、状态、发布时间、提供者和领取者等字段。drift_requests 表保存领取申请，包括申请图书、留言、申请状态、申请时间和申请人信息。

通过这两张表，系统能够实现发布、申请、审核、领取和记录流转结果的完整流程。

### 4.5 权限设计

系统通过 Session 保存当前用户的 account_id、account_type 和 role。学生和教师登录后可以访问图书查询、借阅、座位预约、图书漂流、AI 智能搜索和个人中心等功能。管理员角色可以访问图书维护、用户管理、学生导入、评价审核、互动数据管理、数据大屏和报告生成等功能。

在后端路由中，系统对关键操作进行权限判断。例如添加图书、删除图书、管理用户、审核评价和查看数据大屏等功能仅允许管理员访问；归还图书和释放座位时，系统会判断当前用户是否为对应记录的所有者，从而避免越权操作。

## 第 5 章 系统详细设计与实现

### 5.1 应用初始化与蓝图注册

系统入口文件创建 Flask 应用实例，并通过应用工厂函数完成配置加载、数据库初始化和蓝图注册。系统将不同业务功能拆分为多个蓝图，包括首页模块、认证模块、图书模块、座位模块、AI 工具模块和图书漂流模块。通过蓝图划分，系统路由结构更加清晰，不同模块之间职责明确，便于后期维护。

系统启动时会调用数据库初始化方法创建数据表，并检查部分字段是否存在，以兼容已有数据库结构。同时，系统会初始化默认出版社和示例教师账号，便于系统初次运行和测试。

### 5.2 用户登录与认证实现

登录模块支持学生、教师和管理员等账号统一登录。用户提交账号和密码后，系统首先在学生表中按照学号查询；如果匹配失败，再在教师表中按照工号查询；最后在用户表中按照管理员账号或兼容账号查询。账号存在且密码验证通过后，系统将账号 ID、账号类型、用户名、角色和头像写入 Session，用于后续权限控制和页面展示。

注册模块面向学生和教师两类读者用户。学生注册时填写学号、姓名、性别、专业和密码，系统校验学号是否已存在后写入 students 表；教师注册时填写工号、姓名、性别、专业和密码，系统校验工号是否已存在后写入 teachers 表。密码存储采用 Werkzeug 提供的哈希函数，注册、导入、重置和修改密码时均保存加密后的密码摘要，而不是保存明文密码。修改密码功能会验证旧密码，并检查新密码长度、字母和数字要求，提高账号安全性。

管理员可以通过表格批量导入学生信息，系统支持 xlsx、xls 和 csv 格式文件，并要求表格包含学号、姓名、性别和专业四列。导入时系统会跳过已存在的学号，避免重复插入，并为新导入学生设置默认初始密码。

### 5.3 图书管理功能实现

图书管理模块是系统核心模块之一。管理员可以通过表单录入图书信息，包括 ISBN、书名、作者、出版社、分类、馆藏位置、简介和封面图片。系统在保存图书前会对 ISBN 进行校验，支持 ISBN-10 和 ISBN-13，校验通过后再写入数据库。如果 ISBN 已存在，系统会提示添加失败，避免馆藏数据重复。

图书列表页面支持关键字搜索和分类筛选。用户可以通过书名、作者或 ISBN 进行模糊查询，也可以按分类查看图书。图书详情页面展示图书基本信息、馆藏位置、借阅状态、收藏状态、平均评分和审核通过的评价内容。系统还提供独立的推荐页面，用户可以查看根据个人偏好、全馆借阅热度、收藏热度和最新入库时间生成的推荐图书。

管理员可以编辑图书信息并上传新的封面图片。删除图书时，系统会检查图书是否处于可借状态。如果图书正在被借出，则不允许删除，避免破坏借阅记录与图书状态的一致性。

### 5.4 借阅与归还功能实现

用户在图书列表或详情页面可以借阅状态为可借的图书。借阅时，系统首先查询图书是否存在并判断状态是否为 available。如果图书可借，系统创建一条借阅记录，并根据当前账号类型写入 user_id、student_id 或 teacher_id，同时将图书状态更新为 borrowed。

用户可以在“我的借阅”页面查看个人借阅记录。归还图书时，系统根据记录 ID 查询借阅记录，并判断当前用户是否为该记录所有者。权限校验通过后，系统将借阅记录状态改为 returned，记录归还时间，并将图书状态恢复为 available。

这种设计保证了借阅记录和图书状态同步更新，使系统能够准确反映馆藏图书当前是否可借。

### 5.5 座位预约功能实现

座位预约模块用于管理图书馆学习座位。系统按照楼层和区域展示座位，并在页面中加入入口、服务台、窗边、中心通道、书架和电子阅览区等空间参照信息，使用户能够直观了解座位所在位置。每个座位包含座位号、楼层、区域、是否有电源和当前状态等信息。

用户预约座位前，系统会检查当前用户是否已经存在有效预约。如果存在有效预约，则不允许再次预约，避免同一用户占用多个座位。如果目标座位状态为空闲，系统创建预约记录并将座位状态改为 occupied。

系统还实现了自动释放超时座位的逻辑。每次进入座位首页时，系统会检查所有 active 状态且开始时间超过 3 小时的预约记录，将这些记录状态改为 completed，写入结束时间，并将对应座位状态恢复为 free。对于即将到期的预约，系统会计算剩余时间并在页面提示用户。

### 5.6 图书收藏与评价功能实现

系统为用户提供图书收藏功能。用户可以在图书详情或列表页面收藏图书，系统会在 book_favorites 表中保存收藏记录。用户也可以进入“我的收藏”页面查看所有收藏图书，并随时取消收藏。

评价功能与借阅流程关联。系统要求用户必须归还某本图书后才能评价该图书，并且每个用户对同一本书只能评价一次。评价内容包含 1 至 5 星评分和文字内容，提交后状态为 pending。管理员在互动管理页面审核评价，可以将评价状态改为 approved 或 rejected。只有审核通过的评价才会展示在图书详情页面。

### 5.7 图书漂流功能实现

图书漂流模块用于实现校园闲置书籍共享。用户可以发布漂流图书，填写书名、关联课程、新旧程度、交换方式和补充说明。发布后，图书进入 drifting 状态，其他用户可以在漂流广场中按状态、课程或关键词进行筛选。

用户查看漂流图书详情后，可以提交领取申请并填写留言。系统会检查申请人是否为图书提供者，防止用户申请自己发布的图书；同时检查是否已经申请过同一本书，避免重复申请。

图书提供者或管理员可以处理申请。若同意某个申请，系统会将该申请状态设为 accepted，将漂流图书状态设为 claimed，并记录领取者信息，同时将同一本书的其他申请设为 rejected。若拒绝申请，则只更新该申请状态。管理员还可以编辑和删除漂流图书及相关申请记录。

### 5.8 个性化图书推荐功能实现

系统新增了智能图书推荐页面，用于根据用户行为和全馆数据生成多维度推荐书单。推荐功能首先统计当前用户收藏图书和借阅图书中出现频率较高的分类，形成用户偏好分类；随后排除用户已经借阅或收藏过的图书，避免推荐重复内容。

在推荐结果组织上，系统将推荐书单划分为四类：第一类是根据用户常借阅或收藏分类生成的个性化推荐；第二类是按照全馆借阅次数排序得到的借阅榜推荐；第三类是按照读者收藏次数排序得到的收藏热度推荐；第四类是在用户行为数据不足时展示的最新入库图书推荐。通过这种方式，系统既能利用个人历史行为提高推荐针对性，也能利用全馆热度数据和新书数据保证推荐内容的丰富性。

### 5.9 AI 智能检索与数据看板实现

AI 智能检索模块通过读取本地数据库中的馆藏图书信息，构造包含书名、作者、分类和状态的上下文，再结合用户自然语言问题调用 DeepSeek API。系统提示词要求 AI 只能推荐本地馆藏中真实存在的图书，避免生成虚构图书。该功能使用户无需精确输入书名，也可以通过兴趣、课程或阅读目标获得推荐。

数据看板模块主要面向管理员。系统统计近 30 天借阅趋势、图书分类借阅分布、座位预约热力数据、馆藏总数、累计借阅数、当前借阅数、座位占用率等指标，并生成简短的管理建议。管理员可以通过这些数据了解图书馆使用情况，为补充热门分类图书和优化座位管理提供依据。

### 5.10 个人中心与首页看板实现

个人中心用于展示当前用户的个人使用概况，包括最近借阅记录、当前借阅数量、已归还数量、收藏数量、评价数量、当前预约座位、发布漂流图书数量和申请数量等。用户也可以在个人中心修改密码和上传头像。

首页看板展示系统整体统计数据，包括馆藏总数、今日借阅数量、空闲座位数量和最近活动。最近活动由借阅记录和座位预约记录组合生成，能够让管理员或用户快速了解系统近期使用情况。

#### 6.3.3 图书借阅与归还测试

测试内容：用户预约一个空闲座位，再尝试预约第二个座位，最后释放当前座位。

测试结果：系统能够正确限制评价条件，并实现评价审核流程。

#### 6.3.6 图书漂流测试

测试内容：用户发布漂流图书，其他用户提交申请，发布者同意申请。

测试结果：系统能够完成漂流图书发布、申请和处理流程。

## 第 7 章 总结与展望

### 7.1 总结

本文设计并实现了一套基于 Flask 的智慧图书馆管理系统。系统以校园图书馆实际业务为背景，围绕图书管理、借阅管理、座位预约、用户管理、图书漂流、评价互动和数据分析等需求进行设计。系统采用 Flask 框架实现后端业务逻辑，使用 SQLAlchemy 操作 MySQL 数据库，并通过 Jinja2 模板完成前端页面渲染，同时引入 ECharts 图表库实现数据可视化展示，调用 DeepSeek 大模型接口实现智能检索与内容审核。

在架构设计方面，系统采用蓝图机制组织路由模块，将认证、图书、座位、漂流和 AI 工具等功能解耦为独立的蓝图组件，便于后续维护与扩展。在账户体系方面，系统设计了学生、教师和普通用户三种账户类型，针对借阅记录、座位预约、图书收藏和书评等业务表采用多态外键方案，通过统一的辅助函数实现对不同账户类型的透明操作，兼顾了数据完整性与代码复用性。在权限控制方面，系统基于会话中存储的角色信息，在路由层面对管理员功能和普通用户功能进行区分，确保敏感操作仅对授权用户开放。

在功能实现方面，系统覆盖了图书馆管理的核心业务场景。图书管理模块支持管理员对图书进行增删改查，并引入 ISBN 校验算法确保书目数据的规范性；借阅管理模块实现了借阅、归还及个人借阅记录查询的完整闭环；座位预约模块通过三层可视化布局展示座位状态，支持选座、释放和超时自动回收机制；用户管理模块支持管理员查看全系统账户、重置密码及删除用户；图书漂流模块实现了闲置书籍的发布、认领申请与处理流程，管理员可对全平台漂流数据进行统一管理；批量导入模块支持管理员通过上传表格文件一次性导入大量学生或教师账户，解析过程中自动跳过重复和无效数据；评价互动模块支持用户对已归还图书进行评分与评价，管理员可借助 AI 接口对评论进行自动审核；推荐模块基于用户的借阅和收藏偏好进行个性化图书推荐；数据分析模块通过聚合查询生成借阅趋势、分类偏好和座位使用热力图等可视化图表，并结合 AI 生成运营分析总结。

通过本系统的设计与实现，进一步加深了对 Web 应用开发全流程的理解，包括 Flask 框架的请求响应机制、SQLAlchemy 的 ORM 查询与事务管理、数据库表结构设计与外键约束策略、基于会话的认证与权限控制方案、多类型账户的统一建模方法，以及前后端数据交互与可视化渲染技术。系统整体功能较完整，能够满足校园图书馆信息化管理的基本需求，为后续功能扩展和性能优化奠定了良好基础。

### 7.2 展望

由于时间和开发条件限制，系统仍存在一些可以优化和扩展的方向：

1. 增加分页和高级筛选功能，提高大量图书和记录数据下的浏览效率。
2. 增加借阅期限、续借、逾期提醒和罚款统计等更完整的流通管理功能。
3. 增加座位预约时间段选择和预约取消规则，使座位管理更加精细化。
4. 增加操作日志和系统审计功能，方便管理员追踪关键操作。
5. 优化前端页面响应式布局，提高移动端访问体验。
6. 对图书推荐功能进行进一步优化，引入更细致的用户画像、评分权重和协同过滤算法，提高个性化推荐效果。
7. 增加数据导出、图表交互和管理员报表下载功能，提高管理决策支持能力。

未来可以在现有系统基础上继续扩展功能，使其逐步发展为更加完善、智能和易用的校园智慧图书馆服务平台。
