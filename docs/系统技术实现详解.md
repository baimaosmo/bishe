### 3.3 密码安全

三个用户模型都实现了 `set_password()` 和 `check_password()` 方法：

```python
# models.py 中的密码方法
def set_password(self, password):
    self.password_hash = generate_password_hash(password)  # Werkzeug 哈希

def check_password(self, password):
    return check_password_hash(self.password_hash, password)  # 验证
```

密码从不以明文存储，数据库中只有 `password_hash` 字段。

```python
# Book 删除时自动删除关联的收藏和评价（cascade='all, delete-orphan'）
```

## 四、认证模块（/auth）

> 文件：`app/routes/auth.py` | 蓝图：`auth_bp`

### 4.1 核心工具函数

#### `current_account()` — 获取当前登录用户对象

```python
def current_account():
    if session.get('account_type') == 'student':
        return Student.query.get(session.get('account_id'))      # SELECT * FROM students WHERE id=?
    if session.get('account_type') == 'teacher':
        return Teacher.query.get(session.get('account_id'))      # SELECT * FROM teachers WHERE id=?
    return User.query.get(session.get('account_id'))              # SELECT * FROM users WHERE id=?
```

**数据库操作**：根据 `session` 中的类型标记，用主键查询对应的表。这是 Flask 中典型的"无状态认证"模式——每次请求通过 Cookie 中的 Session ID 反查服务端 Session 数据。

#### `validate_password_strength(password)` — 密码强度校验

```python
def validate_password_strength(password):
    if len(password or '') < 8:                    # 长度 ≥8
        return '密码长度不能少于 8 位。'
    if not re.search(r'[A-Za-z]', password):       # 至少 1 个字母
        return '密码必须包含至少 1 个字母。'
    if not re.search(r'\d', password):              # 至少 1 个数字
        return '密码必须包含至少 1 个数字。'
    if password in {'12345678', 'password', 'Password1'}:  # 黑名单
        return '密码过于简单...'
    return None  # 校验通过
```

#### `save_uploaded_avatar(file)` — 保存头像文件

**完整调用链**：
```
用户点击上传 → POST /auth/upload_avatar
  → request.files.get('avatar')           # 从请求中获取文件对象
  → save_uploaded_avatar(file)             # 调用保存函数
      → secure_filename(file.filename)     # 清理文件名（去掉路径穿越字符）
      → filename.rsplit('.', 1)[1].lower() # 提取扩展名
      → 检查是否在 {'png','jpg','jpeg','gif','webp'} 白名单
      → uuid4().hex                        # 生成唯一文件名
      → os.makedirs(dir, exist_ok=True)    # 确保目录存在
      → file.save(path)                    # 写入磁盘
      → return 'uploads/avatars/xxx.jpg'   # 返回相对路径
  → account.avatar = avatar                # 写入数据库
  → session['avatar'] = avatar             # 同步更新 Session
  → db.session.commit()                    # 提交事务
```

### 4.2 登录流程 — `POST /auth/login`

```
用户提交表单（account + password）
    │
    ▼
策略1：Student.query.filter_by(student_no=account).first()
    │   → 找到 且 check_password(password) 通过？
    │   → 是：session['account_type']='student'，跳转图书列表
    │   → 否：继续
    ▼
策略2：Teacher.query.filter_by(job_no=account).first()
    │   → 找到 且 check_password(password) 通过？
    │   → 是：session['account_type']='teacher'，跳转图书列表
    │   → 否：继续
    ▼
策略3：User.query.filter_by(username=account).first()
    │   → 找到 且 check_password(password) 通过？
    │   → 是：session['account_type']='user'，role=user.role
    │   → 否：flash('账号密码错误')
    ▼
Session 写入字段：
    session['account_id']   = 用户主键
    session['account_type'] = 'student'|'teacher'|'user'
    session['username']     = 显示名称
    session['role']         = 'admin'|'student'|'teacher'|'user'
    session['avatar']       = 头像路径
```

**数据库查询**：每次登录最多执行 3 次 `SELECT`（分别查三张表）+ 3 次 `check_password()`（Python 内存计算，不访问数据库）。

### 4.3 注册流程 — `POST /auth/register`

```
表单提交 → 校验 → 创建 Student/Teacher 对象 → db.session.add() → db.session.commit()
```

关键校验步骤：
1. `account_type` 必须是 `student` 或 `teacher`
2. 学号/工号在三张表中查重（**3 次 SELECT**）
3. 两次密码一致
4. 密码强度校验（`validate_password_strength`）
5. 写入数据库时用 `try-except IntegrityError` 兜底（处理并发重复注册）

### 4.4 批量导入 — `POST /auth/import_students`

```
管理员上传 Excel/CSV 文件
    │
    ▼
pandas 读取文件内容为 DataFrame
    ├─ .csv  → pd.read_csv(file, encoding='utf-8-sig')
    │         失败则回退 pd.read_csv(file, encoding='gbk')
    └─ .xlsx → pd.read_excel(file)
    │
    ▼
校验表头（必须包含"学号/工号、姓名、性别、专业"）
    │
    ▼
逐行遍历 DataFrame：
    ├─ clean_cell() 清洗数据（NaN→空字符串、float→int→str）
    ├─ 检查学号/工号是否已存在 → 存在则 skip_count++
    └─ 不存在 → 创建对象 + set_password() → db.session.add()
    │
    ▼
db.session.commit() 一次性提交所有新记录
```

**数据库操作**：
- 读：每行做一次 `SELECT` 查重（`query.filter_by(...).first()`）
- 写：`db.session.add()` 积累在事务中，最后一次性 `commit()`
- 出错时 `db.session.rollback()` 回滚全部

### 4.5 用户管理路由

| 路由 | 方法 | 功能 | 关键数据库操作 |
|------|------|------|---------------|
| `/auth/manage_users` | GET | 用户列表 | `Student.query.all()` + `Teacher.query.all()` + `User.query.all()` |
| `/auth/reset_password/<type>/<id>` | GET | 重置密码 | `query.get_or_404(id)` → `set_password()` → `commit()` |
| `/auth/delete_user/<type>/<id>` | GET | 删除用户 | `query.get_or_404(id)` → `db.session.delete()` → `commit()`，外键冲突时 catch `IntegrityError` |

**安全措施**：删除用户时禁止管理员删除自己（`if account_type == session['account_type'] and account_id == session['account_id']`）。

## 五、图书管理模块（/books）

### 5.1 核心工具函数

#### `current_account_filter(model)` — 多态查询过滤

```python
def current_account_filter(model):
    """根据当前用户类型，返回对应外键的过滤条件"""
    if session.get('account_type') == 'student':
        return model.student_id == session.get('account_id')
    if session.get('account_type') == 'teacher':
        return model.teacher_id == session.get('account_id')
    return model.user_id == session.get('account_id')

# 使用示例：
records = BorrowRecord.query.filter(current_account_filter(BorrowRecord)).all()
# 等价于：SELECT * FROM borrow_records WHERE student_id=? （如果是学生）
```

这个函数是整个系统的**多态查询核心**：一行代码适配三种账号类型，避免在每个路由中重复写 `if/elif/else`。

#### `assign_current_account(model)` — 多态外键赋值

```python
def assign_current_account(model):
    """根据当前用户类型，设置对应外键字段"""
    if session.get('account_type') == 'student':
        model.student_id = session.get('account_id')
    elif session.get('account_type') == 'teacher':
        model.teacher_id = session.get('account_id')
    else:
        model.user_id = session.get('account_id')

# 使用示例：
new_record = BorrowRecord(book_id=book.id)
assign_current_account(new_record)  # 自动设置 student_id / teacher_id / user_id
db.session.add(new_record)
```

#### `account_owns(record)` — 所有权校验

```python
def account_owns(record):
    """判断当前用户是否拥有该记录（用于归还、释放等权限校验）"""
    if session.get('account_type') == 'student':
        return record.student_id == session.get('account_id')
    if session.get('account_type') == 'teacher':
        return record.teacher_id == session.get('account_id')
    return record.user_id == session.get('account_id')
```

#### `validate_isbn(isbn)` — ISBN 校验位验证

```
输入 ISBN → re.sub(r'[-\s]', '', isbn) 去横线和空格
    ├─ 10位 → 加权和 (权重 10→1) % 11 == 0 ?
    └─ 13位 → 交替加权 (1,3,1,3...) % 10 == 0 ?
返回 (清洗后的ISBN, 错误信息或None)
```

这是数学层面的校验——不仅检查格式，还验证校验位的正确性，防止用户输入虚假 ISBN。

### 5.2 图书列表 — `GET /books/list`

**完整前后端流程**：

```
前端（books/list.html）：
  搜索框 → ?q=关键词
  分类下拉 → ?category=分类
  点击搜索 → GET /books/list?q=Python&category=计算机

后端：
  search_query = request.args.get('q', '').strip()
  category = request.args.get('category', '').strip()

  query = Book.query                              # SELECT * FROM books
  if search_query:
      query = query.filter(                        # WHERE title LIKE '%Python%'
          db.or_(                                  #    OR author LIKE '%Python%'
              Book.title.like(f'%{search_query}%'),#    OR isbn LIKE '%Python%'
              Book.author.like(f'%{search_query}%'),
              Book.isbn.like(f'%{search_query}%')
          ))
  if category:
      query = query.filter(Book.category == category)  # AND category='计算机'

  books = query.order_by(Book.add_time.desc()).all()   # ORDER BY add_time DESC

  同时查询当前用户的收藏 ID 集合（前端显示收藏状态图标）：
  favorite_book_ids = {
      fav.book_id for fav in BookFavorite.query.filter(
          current_account_filter(BookFavorite)
      ).all()

  return render_template('books/list.html', books=books, favorite_book_ids=favorite_book_ids)
```

```
    FROM books LEFT JOIN borrow_records ON borrow_records.book_id = books.id

    GROUP BY books.id
    ORDER BY borrow_count DESC, add_time DESC

    SELECT books.*, COUNT(book_favorites.id) AS favorite_count
    FROM books LEFT JOIN book_favorites ON book_favorites.book_id = books.id
    WHERE books.id NOT IN (排除列表)

    ORDER BY favorite_count DESC, add_time DESC

策略4：最新上架

    ORDER BY add_time DESC LIMIT 8
```

### 6.5 超时自动释放

```python
def cleanup_expired_reservations():
    now = datetime.utcnow()
    expire_threshold = now - timedelta(hours=3)

    expired = SeatReservation.query.filter(
        SeatReservation.status == 'active',
        SeatReservation.start_time <= expire_threshold
    ).all()

    for record in expired:
        record.status = 'completed'
        record.end_time = now
        record.seat.status = 'free'    # 通过 relationship 访问

    if expired:
        db.session.commit()
```

**触发时机**：每次访问座位页面时被动调用。不是后台定时任务。

```
          │   ┌─────────┐  ┌──────────┐
          │   │ claimed │  │ drifting │  ← 书继续漂流
          │   │(已领走) │  └──────────┘
          │   └─────────┘
          │
          └── 其他申请自动变为 rejected
```

### 8.2 核心流程的函数调用链

#### 发布漂流图书
```

    → 绑定提供者：book.provider_student_id = acc_id (三选一)
    → db.session.add(book) → db.session.commit()
```

#### 提交领取申请
```
POST /crossing/request/<book_id>
    → 校验：书是 drifting 状态？
    → 校验：不是自己的书？
    → 校验：没申请过？
    → DriftRequest(book_id, message, status='pending')
    → 绑定申请人 → db.session.add() → commit()
```

#### 处理申请（接受/拒绝）
```
POST /crossing/handle_request/<request_id>/accept

    → 接受：
        dr.status = 'accepted'

        book.receiver_xxx_id = dr.receiver_xxx_id  # 记录领取者
        其他待处理申请 → 全部 status='rejected'     # 自动拒绝
    → 拒绝：
        dr.status = 'rejected'
    → db.session.commit()
```

## 十一、种子数据系统

> 文件：`app/seed_data.py`

### 11.1 幂等设计

所有种子数据函数使用 `first_or_create()` 工具函数：

```python
def first_or_create(model, defaults=None, **filters):
    instance = model.query.filter_by(**filters).first()
    if instance:
        return instance, False       # 已存在，不创建
    params = dict(filters)
    if defaults:
        params.update(defaults)
    instance = model(**params)
    db.session.add(instance)
    db.session.flush()
    return instance, True            # 新创建
```

这意味着 `run.py` 可以反复执行而不会产生重复数据。
