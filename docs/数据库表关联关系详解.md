## 二、三张用户表的定位

`users`、`students`、`teachers` 是**三张完全独立的表**，彼此之间**没有外键关系**。它们在整个系统中扮演"用户基表"的角色，被其他业务表通过多态外键引用。

| 表 | 登录凭证 | 典型用户 | 是否可注册 | 导入方式 |
|------|---------|---------|-----------|---------|
| `users` | `username` | 管理员、校外读者 | ❌（需管理员创建） | ❌ |
| `students` | `student_no` | 校内学生 | ✅ 自助注册 | ✅ Excel/CSV 批量导入 |
| `teachers` | `job_no` | 校内教师 | ✅ 自助注册 | ✅ Excel/CSV 批量导入 |

## 三、外键关系逐表详解

### 3.1 books（图书表）

**自身无外键**，是系统中被引用最多的表。

**被谁引用**：
| 引用方 | 外键字段 | 级联 |
|--------|---------|------|
| `borrow_records` | `book_id` | ❌ |
| `book_favorites` | `book_id` | ✅ 级联删除 |
| `book_reviews` | `book_id` | ✅ 级联删除 |

**关系图**：
```
books (1) ──────< (N) borrow_records     ← 一本书有多条借阅记录
books (1) ──────< (N) book_favorites     ← 一本书可被多人收藏
books (1) ──────< (N) book_reviews       ← 一本书可被多人评价
```

---

### 3.2 borrow_records（借阅记录表）

**4 个外键**：

| 字段 | 指向 | 可空 | 说明 |
|------|------|------|------|
| `user_id` | `users.id` | ✅ NULL | 借阅者是普通用户 |
| `student_id` | `students.id` | ✅ NULL | 借阅者是学生 |
| `teacher_id` | `teachers.id` | ✅ NULL | 借阅者是教师 |
| `book_id` | `books.id` | ❌ NOT NULL | 借了哪本书 |

**三选一规则**：`user_id`、`student_id`、`teacher_id` 每条记录有且仅有一个非空。

**反向引用**（backref）：
```python
user.borrow_records     → 该用户的所有借阅记录
student.borrow_records  → 该学生的所有借阅记录
teacher.borrow_records  → 该教师的所有借阅记录
book.borrow_records     → 该书的所有借阅记录
```

---

### 3.3 seats（座位表）

**自身无外键**（独立字典表）。

**被谁引用**：
| 引用方 | 外键字段 | 级联 |
|--------|---------|------|
| `seat_reservations` | `seat_id` | ❌ |

**关系**：
```
seats (1) ──────< (N) seat_reservations   ← 一个座位有多条预约记录
```

---

### 3.4 seat_reservations（座位预约表）

**4 个外键**：

| 字段 | 指向 | 可空 | 说明 |
|------|------|------|------|
| `user_id` | `users.id` | ✅ NULL | 预约者是普通用户 |
| `student_id` | `students.id` | ✅ NULL | 预约者是学生 |
| `teacher_id` | `teachers.id` | ✅ NULL | 预约者是教师 |
| `seat_id` | `seats.id` | ❌ NOT NULL | 预约了哪个座位 |

**反向引用**：
```python
user.seat_reservations     → 该用户的所有预约记录
seat.reservations          → 该座位的所有预约记录
```

---

### 3.5 publishers（出版社表）

**完全独立的字典表**，无任何外键。与 `books` 表没有 FK 关联——图书的 `publisher` 字段存的是字符串而非外键。出版社仅用于添加/编辑图书时的下拉列表。

---

### 3.6 book_favorites（图书收藏表）

**4 个外键**：

| 字段 | 指向 | 可空 | 说明 |
|------|------|------|------|
| `user_id` | `users.id` | ✅ NULL | 收藏者是普通用户 |
| `student_id` | `students.id` | ✅ NULL | 收藏者是学生 |
| `teacher_id` | `teachers.id` | ✅ NULL | 收藏者是教师 |
| `book_id` | `books.id` | ❌ NOT NULL | 收藏了哪本书 |

**级联删除**：删除图书时，该书的全部收藏记录自动删除。
```python
book = db.relationship('Book', backref=db.backref('favorites', lazy=True,
                           cascade='all, delete-orphan'))
```

**反向引用**：
```python
book.favorites     → 该书被哪些人收藏
user.book_favorites → 该用户收藏了哪些书
```

### 3.7 book_reviews（图书评价表）

**4 个外键**：

| 字段 | 指向 | 可空 | 说明 |
|------|------|------|------|

### 3.8 drift_books（漂流图书表）

**6 个外键**，分两组：

**提供者组**（发布时填入，三选一）：

| 字段 | 指向 | 说明 |
|------|------|------|
| `provider_user_id` | `users.id` | 提供者是普通用户 |
| `provider_student_id` | `students.id` | 提供者是学生 |
| `provider_teacher_id` | `teachers.id` | 提供者是教师 |

**领取者组**（申请被接受后填入，三选一，接受前全为 NULL）：

| 字段 | 指向 | 说明 |
|------|------|------|
| `receiver_user_id` | `users.id` | 领取者是普通用户 |
| `receiver_student_id` | `students.id` | 领取者是学生 |
| `receiver_teacher_id` | `teachers.id` | 领取者是教师 |

**级联删除**：删除漂流图书时，全部关联的 `drift_requests` 自动删除。
```python
requests = db.relationship('DriftRequest', backref='book', lazy=True,
                            cascade='all, delete-orphan')
```

**关系链**：
```
drift_books (1) ──────< (N) drift_requests   ← 一本漂流书可被多人申请
                                │
                                ▼
                          drift_requests (接收申请表)
                                有独立的 receiver_三FK 指向申请者
```

**反向引用**：
```python
provider_user.drift_books_provided      → 该用户提供的漂流书
receiver_user.drift_books_received      → 该用户领到的漂流书
book.requests                           → 该书的申请列表
```

---

### 3.9 drift_requests（漂流图书申请表）

**4 个外键**：

| 字段 | 指向 | 可空 | 说明 |
|------|------|------|------|
| `receiver_user_id` | `users.id` | ✅ NULL | 申请者是普通用户 |
| `receiver_student_id` | `students.id` | ✅ NULL | 申请者是学生 |
| `receiver_teacher_id` | `teachers.id` | ✅ NULL | 申请者是教师 |
| `book_id` | `drift_books.id` | ❌ NOT NULL | 申请了哪本漂流书 |

**反向引用**：
```python
book.requests                → 该书收到的所有申请
receiver_user.drift_requests → 该用户提交的所有申请
```

## 四、多态关联模式总结

系统中有 **6 张业务表** 使用了相同的多态模式：

| 业务表 | 多态外键组 | 含义 |
|--------|-----------|------|
| `borrow_records` | `user_id / student_id / teacher_id` | 谁借的 |
| `seat_reservations` | `user_id / student_id / teacher_id` | 谁预约的 |
| `book_favorites` | `user_id / student_id / teacher_id` | 谁收藏的 |
| `book_reviews` | `user_id / student_id / teacher_id` | 谁评价的 |
| `drift_books` | `provider_user/student/teacher_id` | 谁提供的 |
| `drift_books` | `receiver_user/student/teacher_id` | 谁领走的 |
| `drift_requests` | `receiver_user/student/teacher_id` | 谁申请的 |

**代码层面**：三个通用工具函数屏蔽了多态差异：

```python
current_account_filter(model)   # 生成 WHERE 条件（查自己）
assign_current_account(model)   # 绑定当前用户外键（写入时）
account_owns(record)            # 判断记录所有权（权限校验）
```

## 五、级联删除关系

| 主表 | 级联删除的表 | 触发条件 |
|------|------------|---------|
| `books` | `book_favorites` | 删除图书 → 自动删除所有收藏记录 |
| `books` | `book_reviews` | 删除图书 → 自动删除所有评价记录 |
| `drift_books` | `drift_requests` | 删除漂流书 → 自动删除所有申请记录 |

**不级联的情况**：
- 删除图书**不会**删除 `borrow_records`（借阅历史需要保留）
- 删除用户**不会**级联删除其借阅/收藏/评价记录（由 `IntegrityError` 保护）

## 六、无 FK 的隐式关联

| 关联 | 方式 | 说明 |
|------|------|------|
| `books.publisher` ↔ `publishers.name` | 字符串匹配 | 图书的 publisher 字段是字符串，非 FK。添加图书时传入的出版社名如果在 publishers 表中不存在则自动创建 |
| `book_favorites` ↔ `borrow_records` | 代码逻辑 | 推荐算法中通过集合运算排除已借阅/已收藏的图书 ID |
| `book_reviews` ↔ `borrow_records` | 代码逻辑 | 用户必须归还过该书才能评价（代码中查询 borrow_records 判断） |
